🔌 API Endpoints

Users API

POST /api/users.php?action=register Ready

Register New User

Create a new customer account with email, password, and profile info.

curl -X POST http://localhost/api/users.php?action=register \ -H "Content-Type: application/json" \ -d '{ "email": "user@example.com", "username": "johndoe", "password": "securepass123", "confirm_password": "securepass123", "first_name": "John", "last_name": "Doe" }'
POST /api/users.php?action=login Ready

Login User

Authenticate user and return authentication token.

curl -X POST http://localhost/api/users.php?action=login \ -H "Content-Type: application/json" \ -d '{"email": "user@example.com", "password": "securepass123"}'
GET /api/users.php?action=profile Ready

Get User Profile

Retrieve authenticated user's profile information.

Requires: Authorization token in header

Products API

GET /api/products.php?action=list&page=1&limit=20 Ready

List Products

Get paginated list of all products with optional filtering by category, price, or search term.

Parameter Type Description
page integer Page number (default: 1)
limit integer Items per page (default: 20)
category integer Filter by category ID
min_price decimal Minimum price filter
max_price decimal Maximum price filter
sort string Sort by: name, price, rating
GET /api/products.php?action=search&q=sofa Ready

Search Products

Full-text search across product names and descriptions.

GET /api/products.php?action=get&id=1 Ready

Get Single Product

Retrieve detailed information for a specific product including reviews.

Orders API

POST /api/orders.php?action=create Ready

Create Order

Create a new order with items from cart and shipping information.

Requires: Authentication token

curl -X POST http://localhost/api/orders.php?action=create \ -H "Content-Type: application/json" \ -H "Authorization: Bearer {token}" \ -d '{ "items": [ {"product_id": 1, "quantity": 2}, {"product_id": 3, "quantity": 1} ], "shipping_address": "123 Main Street", "shipping_city": "Kampala", "shipping_country": "Uganda", "shipping_postal": "00256" }'
GET /api/orders.php?action=list Ready

Get User Orders

Retrieve authenticated user's order history with pagination.

Requires: Authentication token

POST /api/orders.php?action=payment&id=1 Ready

Process Payment

Process payment for an order. Integrate with Stripe/PayPal.

Requires: Authentication token

🚀 Getting Started

1. Setup Database

Import the database schema at database/database.sql into MySQL:

mysql -u root prime_home_hub < database/database.sql

2. Configure Connection

Update database credentials in includes/database.php:

define('DB_HOST', 'localhost'); define('DB_USER', 'root'); define('DB_PASSWORD', ''); define('DB_NAME', 'prime_home_hub');

3. Test First API Call

Open your browser and test the products endpoint:

http://localhost/api/products.php?action=list&limit=5

4. Register a User

Use curl to register a test account:

curl -X POST http://localhost/api/users.php?action=register \ -H "Content-Type: application/json" \ -d '{ "email": "test@example.com", "username": "testuser", "password": "Test@1234", "confirm_password": "Test@1234", "first_name": "Test", "last_name": "User" }'

⚠️ Error Handling

All API responses follow a consistent JSON format:

{ "success": true/false, "message": "Operation description", "data": { /* response data */ }, "error": "Error message if failed" }

HTTP Status Codes

  • 200 OK - Request successful
  • 201 Created - Resource created successfully
  • 400 Bad Request - Invalid input or missing fields
  • 401 Unauthorized - Authentication required
  • 403 Forbidden - Insufficient permissions
  • 404 Not Found - Resource not found
  • 405 Method Not Allowed - Wrong HTTP method
  • 500 Internal Server Error - Server error

💬 Support & Troubleshooting

Database Connection Failed?

  • Check credentials in includes/database.php
  • Verify MySQL is running
  • Ensure database prime_home_hub exists

Authentication Issues?

  • Verify token is passed in Authorization header
  • Check session_token in users table is not NULL
  • Re-login if token has expired

Product Not Showing?

  • Check API response: http://localhost/api/products.php?action=list
  • Verify products in database: SELECT * FROM products;
  • Check product status is 'active'
